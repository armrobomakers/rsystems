## Поведение интерфейса

Фронтенд-логика очень небольшая:

- управляет открытием/закрытием мобильного меню
- скрывает/показывает шапку при скролле на десктопе

### Формы, API и авторизация

В текущем лендинге:

- нет `<form>`
- нет `fetch()`
- нет XHR / WebSocket / EventSource
- нет авторизации
- нет токенов
- нет логики отправки лидов на backend

То есть фронтенд **не связан** с backend-шаблонами и не зависит от серверной бизнес-логики.

## Сборка и запуск

### Локальный запуск

Из `package.json`:

- `npm run site` -> `node scripts/site-server.js`

`site-server.js` - простой локальный HTTP-сервер на Node.js, который отдает статические файлы из корня репозитория.

### Docker

- `npm run docker:up` -> `docker compose up --build`
- `Dockerfile` использует `nginx:1.27-alpine`
- `docker-compose.yml` публикует контейнер на `127.0.0.1:8888`
- `nginx.conf` делает `try_files ... /index.html`

### Vercel

Файл [`vercel.json`](./vercel.json) содержит только:

- `cleanUrls: true`
- `trailingSlash: false`

Вывод:

- деплой на Vercel идет как **статический сайт**
- build-шага в репозитории не видно
- отдельного React/Next build pipeline нет
